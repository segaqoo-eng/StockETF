# 便當 5 策略 — 決策紀錄與實戰指南

> 寫給未來的自己（和朋友）看：為什麼選便當 5、它能不能打贏 0050、怎麼跟單。

## TL;DR

- **策略**：分數 > 65 進場，**浮動 +5% 鎖利出場 / -5% 停損**。其他什麼都不管。
- **資金管理**：100 萬本金、同時最多 5 檔（每檔 20 萬）、訊號太多時取分數最高的。
- **半年回測（2025-10 ~ 2026-04，扣 0.5% 來回成本）**：
  - 便當 5 + 5 部位：**+69.11%** 報酬，0050 同期 +52.79%，**超額 +16.31%**
  - 勝率 61.5%，最大回撤 -10.36%
- **風險**：回測樣本期含科技股大多頭，不代表未來。實盤先小資金驗證再加碼。

## 評分模型（沿用 ranking.js / backtest.py）

對每檔 TW 上市股票算 0~100 分，由 4 大組成：

| 元件 | 上限 | 內容 |
|---|---|---|
| 技術面 | ±30 | RSI、MACD 柱狀體、KD 黃金/死亡交叉、MA 5/20/60 排列 |
| 籌碼面 | ±30 | 外資/投信/自營商 5 日淨買賣、外資連續性、三大法人同向 |
| 量價 | ±20 | 量價齊揚/背離、爆量收長黑/長紅 |
| 趨勢 | ±20 | 60 日新高/新低、突破 MA20、單日漲跌過大扣分 |
| | | **基準分 50**，加總後 clamp 到 [0, 100] |

→ 70+ 強買進、60-69 買進、40-59 觀望、20-39 減碼、< 20 強賣出。

## 策略選型實驗

我們把 5 種策略丟進回測一次比過：

| 策略 | 買 | 賣分數 | 停損 | 獲利出場 | 1034 個訊號 vs 5 部位的結果 |
|---|---|---|---|---|---|
| baseline | 65 | 60 | 無 | 無 | +37.11% |
| **強組合** | **75** | **50** | **-8%** | 無 | +46.32% (3 部位 +78.55%) |
| 便當 3 | 65 | 無 | -5% | +3% | +45.28% |
| **便當 5** ⭐ | **65** | **無** | **-5%** | **+5%** | **+69.11%** |
| 嚴便當 | 75 | 無 | -5% | +5% | +41.26% |

**為什麼選便當 5（不選報酬最高的強組合 3 部位 +78.55%）**：

1. **勝率高**（61.5% vs 強組合的 44.8%）— 一半以上交易賺得到，心態穩
2. **操作簡單**：進場後不看分數、只看價，一目了然
3. **TP/SL 明確**：±5%，券商「條件單」直接設好就走人，不用盯盤
4. **容錯空間**：5 部位比 3 部位有一些分散，單一爆雷影響可控
5. **訊號充足**：1034 個訊號 → 130 個成交，不會因「找不到強訊號」而空手

強組合 3 部位雖然報酬最高，但只 29 筆成交、樣本太小、誤差大、勝率 < 50% 心理上難承受。

詳細比較見「Phase 3 中繼決策點」一節。

## 為什麼這個結果可信？

1. **扣交易成本**：每筆來回 0.5%（手續費 0.1+0.1% + 證交稅 0.3%）
2. **延長到 6 個月**：含上漲、修正、橫盤多種市場狀態（雖然整體偏多頭）
3. **0050 基準對照**：對比同期被動投資結果
4. **資金管理約束**：5 部位限制，貼近真實能執行的場景
5. **排除最簡單的解**：BUY threshold 升到 75 反而幫不大忙（嚴買 +46%，比 baseline 進步只 9%），主因是「出場條件」不是「進場條件」

## 為什麼這個結果**不可信**？

1. **樣本期短**（半年）— 不含真正空頭。2026 年若崩盤，回測結果無參考價值
2. **倖存者偏差**：用今天 ETF 持股當回測標的，這些股票本來就「過去半年表現好」才被 ETF 納入
3. **滑價未考量**：實際進出場成交價可能偏離回測收盤價 0.5-1%
4. **數據源**：FinMind / yfinance 都是 EOD 資料，盤中分數變化未模擬
5. **出場觸發**：backtest 用「明日收盤」執行 TP/SL，現實中條件單觸發時間早於收盤

## 實戰流程

### 每日（建議晚上 8 點以後跑）

1. **更新資料 + 跑回測**
   ```
   雙擊 update.bat
   ```
   或設 Task Scheduler 每天 14:00 自動跑（見 `docs/AUTO_REFRESH_SETUP.md`）

2. **看 status_today.md**
   - 「📋 今日重點」一眼看出今日該做幾筆進/出場
   - 「🟢 最新進場訊號」取分數最高的 **前 5 檔**
   - 「✅ 達標出場 / ⛔ 停損出場」現有部位該出的清單
   - 「⏳ 持倉中」TP/SL 委託價對照

3. **券商下單**
   - 隔日開盤後執行進場（市價或開盤限價）
   - 立即設 +5% TP 條件單 + -5% SL 條件單
   - **不用盯盤**，被動等觸發

4. **記錄到 my_positions.json**
   ```
   python scripts/my_status.py add 7769 鴻勁 5220 1000 2026-04-30
   python scripts/my_status.py close 7769 5481 2026-05-03
   ```
   下次 update.bat 跑完，my_status.md 會自動更新 P&L、回本進度

### 週末 / 月底

- 看 my_status.md 累計損益、回本進度
- 看 backtest_results.csv 最新數字、確認策略仍有效
- 跑 `python scripts/portfolio_sim.py` 看不同策略 / 部位數的表現

### 出狀況時

| 症狀 | 處理 |
|---|---|
| 連續 3 筆停損 | 暫停一週、檢查市場是否進入空頭 |
| 月度報酬 < 0050 | 接下來一個月只跟 0050 |
| 季度報酬 < 0050 | 重做策略選型實驗，可能要改變數 |
| update.bat 拿不到資料 | 看 `data/finmind_cache/` / DB 是否累積，註冊 FinMind token |

## 風險紀律

1. **單檔 ≤ 20% 倉位** — 5 部位限制就是這個用意
2. **總倉位 ≤ 80% 資金** — 留 20% 現金應急
3. **連虧三檔暫停** — 心態壞了不要再下單
4. **不重押任何熱門股** — 即使分數 95，也只買 1/5 倉位
5. **每月對帳** — 實際結果與 backtest 偏差 > 30% 要重新檢視
6. **不超出本金** — 永遠不借錢、不開融資、不玩當沖
